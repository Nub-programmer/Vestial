# Vestial: AI-Powered Company Intelligence Platform

A production-grade SaaS application for instant company research and financial intelligence. Search for any company by name or ticker and get AI-powered summaries, market data, news analysis, risks, and opportunities—all in minutes.

## 🎯 Overview

Vestial revolutionizes how people research companies by combining:
- **Real-time market data** from Finnhub (prices, market cap, P/E ratios, 52-week ranges)
- **Latest news feeds** from NewsAPI with sentiment analysis
- **AI-generated insights** from Groq (professional summaries and beginner explanations)
- **Smart analysis** of risks, opportunities, and bullish/bearish factors
- **Intuitive UI** built for both beginners and experienced investors

Perfect for students learning finance, retail investors, career researchers, and anyone curious about companies.

## 🚀 Quick Start

### Prerequisites
- Node.js 18+ and npm/yarn/pnpm
- PostgreSQL database (Neon recommended for free tier)
- Free API keys from:
  - [Groq Console](https://console.groq.com) - for AI summaries
  - [NewsAPI](https://newsapi.org) - for news data
  - [Finnhub](https://finnhub.io) - for market data

### Installation

```bash
# Clone the repository
git clone <repository-url>
cd vestial

# Install dependencies
npm install

# Copy environment file
cp .env.example .env.local

# Add your API keys to .env.local
```

### Configuration

Edit `.env.local` with your API keys:

```
GROQ_API_KEY=your_groq_api_key_here
NEWSAPI_KEY=your_newsapi_key_here
FINNHUB_API_KEY=your_finnhub_api_key_here
DATABASE_URL=postgresql://user:password@host/vestial
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

### Running Locally

```bash
# Development server (with hot reload)
npm run dev

# Open http://localhost:3000 in your browser
```

The app will start with fallback data if API keys are not configured, allowing you to test the UI immediately.

### Database Setup (Optional)

To enable watchlist and comparison history:

```bash
# Create Neon PostgreSQL database at https://neon.tech (free tier available)

# Push schema to database
npm run db:push

# View database in Prisma Studio
npm run db:studio
```

## 🔐 Security & Environment Variables

**Never commit `.env.local` to version control!**

All API keys are kept server-side and never exposed to the client:
- Environment variables are only read on the server
- Client-side code uses `/api` endpoints
- No sensitive data in browser console or network requests

To deploy:
1. Set environment variables in Vercel dashboard
2. Use Vercel's **Environment Variables** settings
3. No need to modify code—automatically injected at build time

## 🚢 Deployment to Vercel

### Step 1: Prepare Repository
```bash
# Make sure all changes are committed
git add .
git commit -m "Ready for deployment"
git push origin main
```

### Step 2: Connect to Vercel
1. Visit [vercel.com](https://vercel.com)
2. Click "New Project"
3. Select your GitHub repository
4. Click "Import"

### Step 3: Environment Variables
1. In Vercel dashboard, go to **Settings** → **Environment Variables**
2. Add all variables from `.env.example`:
   - `GROQ_API_KEY`
   - `NEWSAPI_KEY`
   - `FINNHUB_API_KEY`
   - `DATABASE_URL` (Neon connection string)
   - `NEXT_PUBLIC_SITE_URL` (your production URL)
3. Click "Save"

### Step 4: Deploy
Click "Deploy" and wait for the build to complete. Your app will be live at `your-project.vercel.app`.

### Database for Production
1. Create a Neon PostgreSQL database: [neon.tech](https://neon.tech)
2. Get the connection string (with password)
3. Add `DATABASE_URL` to Vercel environment variables
4. Run migrations:
   ```bash
   npm run db:push
   ```

## 🎯 Usage Examples

### Search for a Company
1. Go to `/search` or click "Start Exploring"
